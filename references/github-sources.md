# GitHub Research Basis

MixPPT Kim synthesizes generally applicable workflow and design ideas from the projects below. It includes no copied third-party source code, claims no affiliation with upstream authors, and delegates artifact generation to skills already installed in the user's Codex environment.

## Sources

- [ningzimu/codex-ppt-skill](https://github.com/ningzimu/codex-ppt-skill) — staged outline, visual-direction selection, and representative sample approval before full generation. Check the upstream repository for its current license.
- [sirilsengolraj-source/presentation-skill](https://github.com/sirilsengolraj-source/presentation-skill) — editable deck workspaces, redesign flow, rendering, and repeatable QA. Check the upstream repository for its current license.
- [Gabberflast/academic-pptx-skill](https://github.com/Gabberflast/academic-pptx-skill) — action titles, title-only argument testing, exhibit discipline, and citation standards; reported by the repository as MIT licensed at research time.
- [tristan-mcinnis/pptx-from-layouts-skill](https://github.com/tristan-mcinnis/pptx-from-layouts-skill) — semantic use of actual PowerPoint template layouts and masters. Check the upstream repository for its current license.
- [gitbrent/PptxGenJS](https://github.com/gitbrent/PptxGenJS) — editable OOXML generation, charts, tables, media, and slide masters. Check the upstream repository for its current license.
- [presenton/presenton](https://github.com/presenton/presenton) — paired editable PPTX/PDF export, templates, and provider-neutral generation; reported by the repository as Apache-2.0 licensed at research time.
- [tfriedel/claude-office-skills](https://github.com/tfriedel/claude-office-skills) — coordinated PowerPoint and PDF document workflows. Check the upstream repository for its current license.

## Adaptation policy

Only high-level procedures and broadly applicable presentation principles are adapted. Before copying any future upstream code, templates, icons, or other assets, inspect and comply with that repository's current license and attribution requirements.
